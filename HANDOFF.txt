목적
----
이 프로젝트는 개인용 로컬 가계부 앱이다.

현재 앱은 Vue 3 + Vite + TypeScript + Pinia + Vitest 기반의 PWA이다.
데이터는 서버/클라우드 없이 브라우저 IndexedDB에 저장하며, JSON 내보내기/가져오기로 수동 백업/복원을 지원한다.
로그인, 클라우드 동기화, 은행/카드 자동 연동, 백엔드 서버는 아직 범위 밖이다.

- 과한 구조보다 작게 시작하고 나중에 확장 가능한 구조를 선호한다.

현재 앱 구조
------------
상단 탭은 4개다.

1. 입력
   - 월 수입 저장.
   - 일일 지출 입력.
   - 금액 입력은 콤마 표시.
   - 저장 후 input 정리.

2. 대시보드
   - 등록 년도 선택.
   - 선택 년도에 등록된 월 칩 표시.
   - 월 칩 클릭 시 해당 월 상세 내용 표시.
   - 월 수입, 지출, 남은 금액, 지출률 표시.
   - 카테고리별 지출과 월별 지출 목록 표시.

3. 통계
   - 년도별 총 지출 CSS 막대 그래프.
   - 선택 년도의 월별 지출 CSS 막대 그래프.
   - 총 지출, 지출월 평균, 최고 지출 월 요약 카드.
   - `지출 항목` 콤보박스로 전체/카테고리별 통계 필터링.
   - 필터 제목과 콤보박스는 우측 정렬됨.
   - 선택 항목에 지출이 없으면 `선택한 항목의 지출 기록이 없습니다.` 표시.
   - 외부 차트 라이브러리는 사용하지 않음.

4. 사람
   - 사람별 받을 돈/줄 돈 기록.
   - 정산 완료 기록은 히스토리에 남김.
   - 정산 완료 기록은 현재 미정산 잔액에서만 제외.


저장 및 PWA 구조
----------------
- production store는 `IndexedDbBudgetRepository`를 사용한다.
- IndexedDB:
  - DB 이름: `local-budget-app`
  - object store: `budget`
  - key: `current`
  - 저장 값: `BudgetData` 전체 스냅샷
- localStorage 자동 이관은 하지 않는다. 사용자가 기존 데이터는 테스트 데이터라서 자동 이관 불필요하다고 결정했다.
- JSON 백업/복원은 유지한다.
- 저장 액션은 저장소 save 성공 후에만 store 메모리 상태를 커밋한다.
- import 저장 실패 시 가져온 데이터가 화면/store에 남지 않도록 테스트가 있다.
- `initialize()`는 동시 호출에 대해 idempotent이며, load 실패 후 재시도 가능하다.
- PWA 자산:
  - `public/manifest.webmanifest`
  - `public/sw.js`
  - `public/icons/icon-192.svg`
  - `public/icons/icon-512.svg`
- `src/main.ts`에서 `/sw.js` service worker를 window load 후 등록한다.
