# 캘린더 탭 설계

## 목표

`캘린더` 탭을 추가해서 선택한 월의 일자별 지출을 달력 형태로 보여준다.

사용자는 년도와 월을 콤보박스로 선택하고, 각 날짜에 등록된 지출을 카테고리명과 금액으로 확인한다. 날짜를 클릭하면 입력 탭으로 이동하며 해당 날짜가 지출 날짜로 자동 선택된다.

## 선택한 방향

이번 구현은 **월간 캘린더 조회 + 날짜 클릭 입력 이동**으로 진행한다.

- 캘린더의 년월 선택은 기존 `store.selectedMonth`와 공유한다.
- 달력은 일요일 시작 7열 구조로 표시한다.
- 날짜 숫자는 각 셀의 상단 왼쪽 구석에 배치한다.
- 주말 날짜 숫자와 `일`, `토` 요일 헤더는 빨간색으로 표시한다.
- 날짜 셀은 지출이 없어도 클릭 가능하다.
- 지출명은 메모가 아니라 카테고리명 기준으로 표시한다.

## 데이터 흐름

```text
IndexedDB
  ↓
Pinia budgetStore.data.expenses
  ↓
CalendarTab에서 선택 월 기준으로 필터링
  ↓
날짜별 지출 그룹 생성
  ↓
캘린더 셀에 카테고리명과 금액 표시
```

저장 구조 변경은 없다. 기존 `Expense`의 `date`, `month`, `categoryId`, `amount`를 사용한다.

## 사용자 흐름

1. 사용자가 `캘린더` 탭을 클릭한다.
2. 년도와 월 콤보박스가 표시된다.
3. 선택한 월의 날짜가 캘린더로 나열된다.
4. 날짜별 지출이 카테고리명과 금액으로 표시된다.
5. 하루 지출이 4건 이상이면 앞 3건을 표시하고 3번째 항목에 `외 N건`을 붙인다.
6. 사용자가 날짜 셀을 클릭한다.
7. 앱은 입력 탭으로 이동하고 지출 날짜 필드에 선택 날짜를 넣는다.
8. 상단 상태 메시지로 `YYYY-MM-DD 날짜로 지출 입력을 준비했습니다.`를 표시한다.

## 표시 예시

```text
7

점심/외식 5,000원
데이트/여가 40,000원
선물/경조사 10,000원 외 1건
```

## 컴포넌트 변경

### `src/App.vue`

- 탭 타입에 `calendar`를 추가한다.
- 탭 목록에 `캘린더`를 추가한다.
- `CalendarTab`을 렌더링한다.
- `pendingExpenseDate` 상태를 추가한다.
- `CalendarTab`의 날짜 선택 이벤트를 받아 입력 탭으로 이동한다.

### `src/components/CalendarTab.vue`

새 컴포넌트로 추가한다.

- 년도 select와 월 select를 렌더링한다.
- 선택 변경 시 `store.setSelectedMonth()`를 호출한다.
- 선택 월의 달력 셀을 계산한다.
- 선택 월 지출을 날짜별로 묶는다.
- 날짜 셀 클릭 시 선택 날짜를 emit한다.

### `src/components/LedgerTab.vue`

- `initialExpenseDate` prop을 선택적으로 받는다.
- prop 값이 바뀌면 지출 날짜 입력값을 해당 날짜로 바꾼다.
- 해당 날짜의 월을 `store.selectedMonth`에도 반영한다.

### `src/styles.css`

- 탭 5개 레이아웃을 지원한다.
- 캘린더 툴바, 그리드, 날짜 셀, 주말 날짜 스타일을 추가한다.
- 모바일에서도 7열 달력을 유지하되 텍스트가 셀 밖으로 넘치지 않게 한다.
