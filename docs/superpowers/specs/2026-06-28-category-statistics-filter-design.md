# 항목별 통계 필터 설계

## 목표

현재 `통계` 탭은 전체 지출 기준으로 년도별 총 지출과 선택 년도의 월별 지출 흐름을 보여준다.

이번 개선은 통계 탭 상단에 `지출 항목` 콤보박스를 추가해서 사용자가 특정 항목만 골라 통계를 볼 수 있게 하는 것이다.

사용자가 확인해야 하는 내용은 다음과 같다.

- 전체 지출 기준 통계
- 점심/외식, 생활비, 교통 등 특정 항목 기준 통계
- 선택한 항목의 년도별 지출 흐름
- 선택한 항목의 선택 년도 월별 지출 흐름
- 선택한 항목의 총 지출, 지출월 평균, 최고 지출 월

## 선택한 방향

사용자가 선택한 **시안 A: 통계 전체에 항목 필터 적용**으로 진행한다.

상단 콤보박스에서 `전체 지출` 또는 특정 지출 항목을 선택하면 통계 탭의 모든 수치와 그래프가 같은 기준으로 바뀐다.

이 방식은 현재 통계 탭 구조를 가장 적게 바꾸면서도 사용자가 원하는 “항목별로 보기”를 직접 해결한다.

## 사용자 경험

통계 탭 상단에 필터 패널을 추가한다.

- 제목: `통계 기준`
- 콤보박스 라벨: `지출 항목`
- 기본 선택값: `전체 지출`
- 선택 가능 항목:
  - 전체 지출
  - 점심/외식
  - 생활비
  - 고정비
  - 데이트/여가
  - 장보기/식재료
  - 교통
  - 의료/건강
  - 선물/경조사
  - 기타

항목 선택 시 아래 영역이 모두 선택 항목 기준으로 갱신된다.

1. 년도별 지출 그래프
   - `전체 지출` 선택 시 기존처럼 전체 지출 합계를 보여준다.
   - 특정 항목 선택 시 해당 항목의 년도별 지출만 보여준다.

2. 선택 년도 요약 카드
   - 총 지출
   - 지출월 평균
   - 최고 지출 월

3. 월별 지출 흐름
   - 선택 년도의 1월부터 12월까지 표시한다.
   - 특정 항목 선택 시 해당 항목의 월별 지출만 표시한다.

## 빈 상태

지출 데이터가 하나도 없으면 기존 빈 상태를 유지한다.

특정 항목을 선택했는데 해당 항목 지출이 없을 때는 통계 탭 전체를 빈 상태로 바꾸지 않는다.

대신 다음처럼 표시한다.

- 년도별 그래프: 해당 항목 지출이 있는 년도만 표시한다.
- 선택한 항목에 지출이 전혀 없으면 짧은 빈 문구를 보여준다.
- 문구 예시: `선택한 항목의 지출 기록이 없습니다.`

이렇게 하면 전체 통계는 존재하지만 특정 항목만 비어 있는 상황을 자연스럽게 구분할 수 있다.

## 데이터 모델

저장 구조 변경은 필요 없다.

기존 `BudgetData.expenses`의 `categoryId`를 기준으로 필터링한다.

현재 계산 함수는 전체 지출 기준이다.

- `calculateYearlyExpenseStats(expenses)`
- `calculateMonthlyExpenseStats(year, expenses)`

이번 구현에서는 별도 저장 필드 없이, 컴포넌트 또는 스토어에서 필터링된 `expenses`를 이 함수들에 전달하는 방식으로 처리한다.

필터 값 타입은 다음처럼 잡는다.

- `all`: 전체 지출
- `CategoryId`: 특정 지출 항목

## 컴포넌트 구조

주요 변경 대상은 `src/components/StatisticsTab.vue`다.

추가 상태:

- `selectedCategoryFilter`

파생 값:

- `filteredExpenses`
- `filteredYearlyExpenseStats`
- `filteredExpenseYears`
- `filteredMonthlyStats`

스토어에 꼭 추가해야 하는 값이 아니면 `StatisticsTab.vue` 내부 계산으로 시작한다.  
다만 테스트가 복잡해지거나 다른 화면에서도 재사용할 필요가 생기면 계산 유틸로 분리한다.

## 시각 디자인

기존 통계 탭 디자인을 유지한다.

- 필터 패널은 기존 `panel`, `section-heading`, `select` 스타일을 재사용한다.
- 콤보박스는 통계 탭 최상단에 배치한다.
- 그래프 막대 스타일은 기존 년도별/월별 막대 그래프를 그대로 사용한다.
- 새 색상은 추가하지 않는다.
- 모바일에서는 콤보박스가 전체 너비로 내려가도록 한다.

## 테스트

구현은 TDD로 진행한다.

추가할 테스트는 다음 동작을 검증해야 한다.

- 통계 탭에 `지출 항목` 콤보박스가 표시된다.
- 기본값은 `전체 지출`이다.
- 전체 지출 선택 시 기존 전체 통계가 표시된다.
- 특정 항목을 선택하면 년도별 그래프, 요약 카드, 월별 그래프가 해당 항목 기준으로 바뀐다.
- 선택 항목에 지출이 없으면 `선택한 항목의 지출 기록이 없습니다.` 문구가 표시된다.
- 기존 통계 탭 테스트와 대시보드 테스트는 계속 통과해야 한다.

## 이번 범위에서 제외할 것

첫 구현에서는 다음 기능을 넣지 않는다.

- 여러 항목 동시 선택
- 항목별 순위 패널
- 항목 비교 대시보드
- 카테고리별 파이 차트
- 그래프 이미지 내보내기
- 항목 필터 상태를 localStorage에 별도로 저장

이 기능들은 시안 B 또는 C 방향의 후속 확장으로 남긴다.

## 후속 확장

시안 A가 안정화되면 다음 단계로 시안 B의 일부를 추가할 수 있다.

- 선택 년도 항목별 순위
- 전체 지출 흐름 옆에 항목별 지출 랭킹
- 항목 클릭 시 해당 항목 필터 자동 선택

이렇게 하면 사용자는 먼저 전체 흐름을 보고, 많이 쓴 항목을 눌러 상세 흐름으로 들어갈 수 있다.
