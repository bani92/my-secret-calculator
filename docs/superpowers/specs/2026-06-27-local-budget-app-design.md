# 로컬 가계부 앱 설계

## 목표

개인용으로 사용할 간단한 로컬 가계부 웹앱을 만든다. 기존에는 구글 시트에서 월별 시트를 나누고 지출만 기록했지만, 새 앱에서는 월별 흐름을 더 쉽게 보고 입력 부담은 줄이는 것을 목표로 한다.

첫 버전은 브라우저에서 로컬로 실행한다. 로그인, 클라우드 동기화, 결제수단 기록은 포함하지 않는다.

## 기술 방향

첫 버전은 Vue 기반 로컬 웹앱으로 만든다.

사용자가 Java, Spring, Vue를 사용해본 경험이 있으므로 장기적으로는 Spring 백엔드를 붙일 수 있지만, 지금 요구사항은 개인용 로컬 기록 앱이므로 처음부터 서버를 두지 않는다. 데이터는 브라우저 저장소에 저장하고, 나중에 필요해지면 Spring API와 데이터베이스로 확장할 수 있게 데이터 구조를 단순하고 명확하게 유지한다.

권장 첫 버전 기술 스택:

- Vue
- Vite
- TypeScript 또는 JavaScript
- 브라우저 localStorage
- CSS 또는 가벼운 컴포넌트 스타일링

Spring Boot는 다음 상황이 생기면 도입한다:

- 여러 기기에서 같은 데이터를 보고 싶을 때
- 로그인 계정이 필요할 때
- 데이터베이스 저장이 필요할 때
- 백업, 동기화, 권한 관리가 중요해질 때

## 개발 단계

추천 개발 순서는 다음과 같다:

1. Vue 로컬 웹앱을 먼저 만든다.
2. localStorage에 저장하고, 같은 브라우저에서 개인 기록용으로 사용할 수 있게 한다.
3. JSON 내보내기/가져오기 기능을 추가해 수동 백업과 기기 간 데이터 이동을 지원한다.
4. 노트북과 핸드폰의 자동 데이터 공유가 필요해지면 Spring Boot API와 DB를 도입한다.
5. 어디서든 같은 데이터를 쓰려면 Spring Boot 서버와 DB는 노트북 로컬이 아니라 둘 다 접근 가능한 서버나 클라우드 환경에 둔다.

노트북 로컬 DB는 Spring Boot와 DB 연습용으로는 좋지만, 핸드폰과 항상 자동 동기화되는 구조는 아니다. 핸드폰이 노트북 데이터에 접근하려면 노트북 서버가 켜져 있고 같은 네트워크 또는 외부 접속 설정이 필요하다.

## 앱 구조

앱은 두 개의 주요 탭으로 구성한다.

1. 가계부
2. 사람별 돈 기록

가계부 탭은 기본 사용 흐름이다. 월별 수입, 월별 지출, 남은 금액, 카테고리별 합계, 지출 목록을 보여준다.

사람별 돈 기록 탭은 특정 사람과 주고받을 돈을 기록한다. 거래 기록은 계속 보존하되, 정산 완료된 기록은 현재 미정산 금액 계산에서 제외한다.

## 가계부 탭

### 월별 보기

가계부는 월 단위로 관리한다. 구글 시트에서 월별 시트를 나누던 흐름을 앱 안의 월 선택으로 대체한다.

각 월에는 다음 정보를 보여준다:

- 수입 합계
- 지출 합계
- 남은 금액
- 수입 대비 지출률
- 카테고리별 지출 합계
- 선택한 월의 지출 목록

수입이 입력되지 않은 달도 정상적으로 표시되어야 한다.

### 지출 입력

각 지출 기록은 다음 항목을 가진다:

- 날짜
- 카테고리
- 금액
- 메모

첫 버전에서는 결제수단을 기록하지 않는다.

### 기본 카테고리

초기 카테고리는 다음과 같이 시작한다:

- 점심/외식
- 생활비
- 고정비
- 데이트/여가
- 장보기/식재료
- 교통
- 의료/건강
- 선물/경조사
- 기타

화면에는 한국어 카테고리명을 보여준다. 데이터에는 안정적인 내부 ID를 저장해서, 나중에 카테고리 표시 이름을 바꾸더라도 기존 기록이 깨지지 않게 한다.

### 수입 입력

수입은 월 단위로 입력한다. 사용자는 선택한 월에 수입 금액을 하나 입력할 수 있고, 앱은 이 값을 기준으로 남은 금액과 수입 대비 지출률을 계산한다.

수입 입력은 선택 사항이다. 수입이 비어 있어도 지출 기록과 카테고리 합계는 사용할 수 있어야 한다.

## 사람별 돈 기록 탭

### 목적

특정 사람과 돈을 주고받은 기록을 일반 가계부 지출과 섞지 않고 별도로 관리한다.

### 입력 항목

각 사람별 돈 기록은 다음 항목을 가진다:

- 날짜
- 사람 이름
- 방향
- 금액
- 메모
- 정산 상태

방향은 해당 기록이 나에게 어떤 의미인지 나타낸다:

- 받을 돈: 내가 상대에게 받아야 하는 돈
- 갚을 돈: 내가 상대에게 갚아야 하는 돈

### 정산 완료 동작

모든 거래 기록은 정산 완료 후에도 계속 보인다.

기록을 정산 완료로 바꾸면:

- 전체 거래 내역에는 그대로 남는다.
- 현재 미정산 금액 계산에서는 제외된다.
- 목록에서 정산 완료 상태가 보인다.

사람별 요약에는 현재 미정산 금액을 보여준다:

- 양수는 내가 받을 돈이다.
- 음수는 내가 갚을 돈이다.
- 0은 현재 미정산 금액이 없다는 뜻이다.

## 데이터 저장

첫 버전은 브라우저 localStorage에 데이터를 저장한다. 앱을 껐다 켜도 같은 브라우저에서는 데이터가 유지된다.

나중에 내보내기, 가져오기, IndexedDB, Spring 백엔드로 확장할 수 있도록 데이터 구조에는 버전 값을 둔다.

예상 최상위 데이터 구조:

```json
{
  "version": 1,
  "months": {},
  "expenses": [],
  "personRecords": []
}
```

월별 수입은 지출 기록과 분리해서 저장한다. 이렇게 하면 지출 기록 구조가 단순하게 유지된다.

## 화면 원칙

앱은 홍보용 랜딩 페이지가 아니라 실사용 도구처럼 보여야 한다.

중요한 방향:

- 빠른 입력
- 한눈에 보이는 월별 합계
- 카테고리별 지출 확인
- 단순한 탭 구조
- 조용하고 정돈된 화면
- 모바일에서도 쓰기 쉬운 입력 폼과 목록

첫 화면은 소개 페이지가 아니라 실제 가계부 화면이어야 한다.

## 첫 버전 제외 범위

첫 버전에는 다음 기능을 포함하지 않는다:

- 로그인
- 클라우드 동기화
- 결제수단 기록
- 카드나 은행 내역 불러오기
- 복잡한 예산 규칙
- 반복 거래 자동 등록
- 여러 사용자 공유

이 기능들은 로컬 개인 기록 흐름이 충분히 편해진 뒤 다시 검토한다.

## 테스트와 검증

구현 후 다음 항목을 확인한다:

- 지출 추가, 수정, 삭제 시 월별 합계가 갱신된다.
- 월을 바꾸면 해당 월의 지출만 보인다.
- 수입이 비어 있어도 남은 금액과 지출률 표시가 깨지지 않는다.
- 사람별 돈 기록은 정산 완료 후에도 목록에 남는다.
- 정산 완료된 사람별 돈 기록은 현재 미정산 금액에서 제외된다.
- 새로고침 후에도 데이터가 유지된다.
- 데스크톱과 모바일 폭에서 화면이 깨지지 않는다.
